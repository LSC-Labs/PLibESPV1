Detailed Description

template<typename TValue>
class CMultiNameUniqueValueTable< TValue >

Named-value table that allows many names but only unique values.

A value may appear only once in the table. Multiple keys are allowed as long as every key references a different value. set() returns nullptr when a new entry would duplicate an existing value.

This is useful for registries where the same object pointer must not be registered twice, even if callers use different names.
